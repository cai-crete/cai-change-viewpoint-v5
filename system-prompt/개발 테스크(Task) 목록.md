# 📋 [Task] 건축가를 위한 통합 시점 시뮬레이션 시스템 (IVSP-Unified) 개발 태스크 목록

## 1. 프로젝트 초기 세팅 및 환경 구성
- [ ] 1.1. 프론트엔드 및 백엔드 개발 저장소(Repository) 생성 및 초기화
- [ ] 1.2. 프론트엔드 프레임워크 세팅 (예: React, Next.js) 및 기본 라우팅 구성
- [ ] 1.3. 백엔드 API 서버 세팅 (예: Python, FastAPI) 및 기본 구조 작성
- [ ] 1.4. 전역 상태 관리 및 UI 컴포넌트 라이브러리(Tailwind CSS, MUI 등) 설치 및 설정

## 2. 사용자 인터페이스(UI) 개발 (프론트엔드)
- [ ] 2.1. 건축 이미지 업로드 화면 디자인 및 구현 (Drag & Drop 기능 포함)

  - [ ] 2.2.1. 스트리트 뷰(Street View) 선택 버튼
  - [ ] 2.2.2. 조감도(Aerial View) 선택 버튼
  - [ ] 2.2.3. 디테일 뷰(Detail View) 선택 버튼
  - [ ] 2.2.4. 쿼터 뷰(Corner Shot) 선택 버튼
- [ ] 2.3. 시뮬레이션 결과물 출력 및 확인을 위한 뷰어(Viewer) 영역 구현
- [ ] 2.4. 결과물 확인 후 재수정을 요청할 수 있는 피드백(Prompt/옵션) 입력 UI 구현

## 3. 이미지 분석 및 보호 모듈 (백엔드 - Ontological Locking)
- [ ] 3.1. 업로드된 이미지 수신 API 엔드포인트 생성
- [ ] 3.2. 입력 이미지의 현재 카메라 위치(방위각, 고도, 투시) 분석 로직 구현
- [ ] 3.3. 원본 건축물의 매스(Mass), 형태, 주요 재질 속성을 추출하는 로직 구현
- [ ] 3.4. 추출된 데이터를 기하학적 상수로 고정(Locking)하여 변환 방지 데이터 모델 구축

## 4. 광학 시뮬레이션 및 렌더링 엔진 연동 (백엔드 - Viewpoint & Optics)
- [ ] 4.1. 사용자 선택 시점(UI 컨트롤러 값)을 3D 상대 좌표 벡터(Δ)로 변환하는 알고리즘 구현
- [ ] 4.2. 각 시점에 따른 광학 파라미터 매핑 로직 구현
  - [ ] 4.2.1. 스트리트 뷰: 23mm 틸트-시프트 렌즈, 수직 왜곡 보정 세팅
  - [ ] 4.2.2. 조감도: 고도 150m 이상, 32mm 렌즈 세팅
  - [ ] 4.2.3. 디테일 뷰: 110mm 매크로 렌즈, 얕은 피사계 심도 세팅
  - [ ] 4.2.4. 쿼터 뷰: 45mm 표준 렌즈, 2점 투시 세팅
- [ ] 4.3. 사각지대(Blind Spot) 추론 로직 구현 (전면 디자인 DNA 추출 및 후면 배치)
- [ ] 4.4. 변경된 태양광 각도에 따른 재조명(Relighting) 계산 로직 구현
- [ ] 4.5. 위 설정값들을 종합하여 최종 이미지를 생성/렌더링하는 AI 파이프라인 연동

## 5. 결과 반환 및 피드백 루프 처리
- [ ] 5.1. 렌더링 완료된 시뮬레이션 이미지를 프론트엔드로 반환하는 API 구현
